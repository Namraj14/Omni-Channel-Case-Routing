# Presence Status Explained

## What is a Presence Status?

A Presence Status allows agents to indicate whether they are available to receive work from Omni-Channel.

Think of a Presence Status as an agent's current availability state.

Examples:

```text
Available for Cases
Available for Chats
Away
Offline
Busy
```

Omni-Channel checks an agent's Presence Status before assigning work.

---

# Where Presence Status Fits in Omni-Channel

```text
Case
 ↓
Assignment Rule
 ↓
Queue
 ↓
Routing Configuration
 ↓
Omni-Channel
 ↓
Presence Status
 ↓
Agent
```

Even if:

* The Case reaches the Queue
* Routing Configuration exists
* Omni-Channel is enabled

Salesforce will not assign work unless an agent has an appropriate Presence Status.

---

# Why Presence Status is Required

Imagine two support agents:

| Agent | Status              |
| ----- | ------------------- |
| Sarah | Available for Cases |
| John  | Away                |

When a Case enters Omni-Channel:

```text
Case
 ↓
Omni-Channel
 ↓
Available Agents
```

Result:

```text
Sarah receives the Case
```

because she is available.

John is ignored because his status indicates he is unavailable.

# Presence Status Components

## Status Name

### Purpose

The display name visible to agents.

### Example

```text
Available for Cases
```

or

```text
Away
```

or

```text
Busy
```

### Why It Matters

Agents use this status to tell Salesforce whether they are ready to receive work.

---

## Developer Name

### Purpose

The API name generated by Salesforce.

### Example

Status Name:

```text
Available for Cases
```

Developer Name:

```text
Available_for_Cases
```

### Usage

Used internally by:

* Metadata API
* Change Sets
* Deployments
* Integrations

---

# Status Options

A Presence Status must be configured as either:

## Online

### Purpose

Allows agents to receive new work.

### Example

Status:

```text
Available for Cases
```

Option:

```text
Online
```

Result:

```text
Agent receives Cases
```

### Routing Example

```text
Case Created
 ↓
Queue
 ↓
Omni-Channel
 ↓
Available for Cases
 ↓
Agent Receives Work
```

---

## Busy

### Purpose

Prevents agents from receiving new work.

### Example

Status:

```text
Away
```

Option:

```text
Busy
```

Result:

```text
Agent receives no new Cases
```

### Routing Example

```text
Case Created
 ↓
Queue
 ↓
Omni-Channel
 ↓
Away
 ↓
No Assignment
```

---

# Service Channels

## Purpose

Defines which types of work can be delivered while using this status.

A Presence Status can be associated with one or more Service Channels.

---

# Relationship

```text
Presence Status
       ↓
Service Channel
       ↓
Object
```

Example:

```text
Available for Cases
       ↓
Cases Service Channel
       ↓
Case Object
```

This tells Salesforce:

```text
This user can receive Cases
```

# Key Takeaway

A Presence Status answers a simple question:

```text
Can this agent receive work right now?
```

If the status is:

```text
Online
```

and linked to the appropriate Service Channel:

```text
Cases
```

Omni-Channel can route Cases to that agent.

If the status is:

```text
Busy
```

the agent is excluded from routing until they become available again.
